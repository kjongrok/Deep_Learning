  - 로컬 스크립트의 한계를 넘어, 실시간으로 탐지되는 교통량, 밀집도, 이상치 점수를 클라우드 DB에 적재(`log_to_supabase_async`)하는 MLOps 파이프라인 구축.
- [x] **주요 트러블슈팅(Troubleshooting) 내역:**
  1. **[버그] LSTM 차원 불일치 에러 (`ValueError: Expected shape (None, 1, 4)`):** 주피터에서 3차원 LSTM으로 최종 챔피언이 교체되면서 기존 2차원(MLP) 입력 코드와 충돌. ➡️ 입력 배열을 `.reshape(1, 1, 4)`로 명시적으로 3차원으로 펴서 넣도록 복구하여 해결.
  2. **[이슈] 실시간 스트리밍 브라우저 렉(Lag) 현상:** 고화질 이미지를 초당 30번씩 웹으로 쏘면서 브라우저 마비. ➡️ AI는 원본 화질을 보게 하고, 웹 브라우저로 쏠 때만 이미지를 가볍게 리사이징(`cv2.resize`)하며, 초당 3번(`frame_skip=10`)만 업데이트하도록 분할 최적화하여 렉 완벽 제거.
  3. **[이슈] 차량 1대만 지나가도 이상탐지(Anomaly) 알람 발생:** 낮 시간대의 꽉 막힌 도로(20~50대)로만 학습되어, 새벽에 차가 1대만 있어도 이상치(Outlier)로 판단하는 현상. ➡️ `(car+bus+truck) < 5` 일 때는 한산한 도로로 간주하고 무조건 정상 처리하는 도메인 지식(예외 룰) 추가.
  4. **[이슈] 야간 영상 차량 인식률 저하:** 어둠과 전조등 불빛으로 인해 기본 Confidence(0.25)에서 차량을 놓침. ➡️ 파이썬의 `time.localtime().tm_hour`를 읽어들여, 야간(19시~06시)에는 `conf=0.15`로 예민하게, 주간에는 `0.25`로 오탐지를 방지하는 **'주/야간 동적 튜닝(Dynamic Tuning)'** 시스템 창안.
  5. **[이슈] dataset 폴더 및 용량 제한 2GB~5GB으로 인한 먹통 현상:** 배포를 위해 폴더를 비우면서 로컬 파일(.jpg, .csv) 참조 에러 발생. ➡️ 2탭은 즉석에서 아무 비디오나 넣을 수 있는 **'비디오 직접 업로드 기능'**으로 개조하고, 5탭은 죽은 CSV 파일 대신 **'실시간 클라우드 DB 연동(Live Analytics)'**으로 기능을 초월 업그레이드하여 해결.
  6. **[배포] Streamlit Cloud 해외 IP 차단 및 로컬(ngrok) 우회 배포 결정:**
     - 윈도우 전용 패키지 충돌 및 OpenCV 그래픽 모듈 에러 발생 ➡️ `requirements.txt` 최소화 및 `opencv-python-headless` 교체로 대응.
     - 국가교통정보센터 API가 스트림릿(미국) 서버 IP를 차단하여 `HTTPSConnectionPool` 에러 발생 ➡️ 1차적으로 에러 예외 처리 후 가상 데모(Mock Data)를 띄우도록 방어막 코드를 추가하였음.
     - **[최종 의사결정]** 가상 데모 대신 실제 한국 고속도로 CCTV 실시간 분석 결과를 시연하기 위해, **한국 내 로컬 PC에서 구동하고 ngrok(또는 localtunnel)을 통해 외부로 터널링하는 방식**으로 최종 아키텍처를 변경함. 이에 따라 리눅스 서버용 의존성 파일이었던 `packages.txt`는 불필요해져 영구 삭제 처리함.

## 💡 프로젝트 전 구간: 핵심 트러블슈팅 및 의사결정 총정리 (Troubleshooting & Decisions)
Streamlit 도입 이전, 데이터 전처리와 모델 학습 단계부터 겪었던 수많은 시행착오와 해결 과정입니다. 발표 시 **"우리가 얼마나 많은 고민을 거쳐 이 구조를 완성했는가"**를 어필하기 좋은 자료입니다.

1. **[구조] 파편화된 노트북과 스크립트의 난립:**
   - **이슈:** `1_preprocessing.ipynb`, `2_yolo.ipynb`, `train_and_save.py` 등 여러 파일로 나뉘어 있어 전체 파이프라인(데이터 로드 ➡️ 객체 탐지 ➡️ 이상 탐지 ➡️ 저장)을 한눈에 파악하거나 재현하기 매우 어려웠음.
   - **해결:** 모든 과정을 하나의 스토리라인으로 묶은 **`Total_Pipeline.ipynb` 단일 파일 체제**로 완벽히 통합하여 분석 흐름의 가독성을 극대화함.

2. **[환경] 절대경로 하드코딩으로 인한 PC 이동 시 구동 불가 문제:**
   - **이슈:** 코드 곳곳에 `C:\AI_1team\...` 형태의 절대경로가 박혀 있어, 심사위원의 PC나 깃허브 클론 시 무조건 에러가 발생하는 치명적 결함 존재.
   - **해결:** 모든 경로를 `os.path`와 `./models/` 같은 **상대경로(Relative Path)**로 전면 리팩토링하여 어떤 환경, 어떤 드라이브에서도 폴더만 열면 즉시 실행되도록 이식성(Portability)을 확보함.

3. **[모델링] YOLOv8 모델 크기(Nano vs Small vs Medium) 트레이드오프:**
   - **이슈:** 탐지 정확도를 높이기 위해 YOLOv8 Medium 모델을 사용하려 했으나, 실시간 CCTV(초당 30프레임) 처리 시 엄청난 렉과 프레임 드랍 발생.
   - **해결:** 모델별 연산량(FPS)과 mAP를 벤치마크 테스트한 결과, 고속도로의 자동차처럼 크고 뚜렷한 객체는 **YOLOv8 Nano(가장 가벼운 모델)**로도 정확도가 충분히 방어됨을 확인하고 Nano 모델을 채택하여 실시간 관제 속도를 10배 이상 끌어올림.

4. **[모델링] 이상탐지 알고리즘 선정의 딜레마 (K-Means vs MLP vs LSTM):**
   - **이슈:** 초기에는 K-Means 등 군집화(Clustering) 모델을 썼으나 시간의 흐름(Sequence)을 전혀 기억하지 못했고, 딥러닝(MLP) 역시 과거 프레임을 고려하지 않아 오탐지가 잦았음.
   - **해결:** 시계열 데이터의 과거 맥락을 기억하는 **LSTM Autoencoder** 구조를 도입. 특히 뉴런 개수를 8, 16, 32로 늘려가며 실험한 결과, 32-Neuron은 학습 데이터에 과적합(Overfitting)되어 분산이 커지는 현상을 발견함. 결국 가장 밸런스가 좋은 **16-Neuron 모델**을 챔피언으로 선정하여 방어 논리를 확립함.
